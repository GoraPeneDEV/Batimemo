# Politique de Confidentialité — Batimemo

**Application :** Batimemo — Open Core HR
**Identifiant :** com.batimemo.app
**Dernière mise à jour :** 21 février 2026
**Version :** 1.0

## 1. Introduction

Batimemo (ci-après « nous », « notre » ou « la Société ») exploite l'application mobile **Batimemo — Open Core HR**, une solution de gestion des ressources humaines destinée aux employés et aux entreprises abonnées à la suite **Open Core Business Suite**.

La présente Politique de Confidentialité décrit les types de données personnelles que nous collectons, la manière dont nous les utilisons, les partagerons et les protégeons, ainsi que les droits dont vous disposez à leur égard.

En utilisant l'application, vous acceptez les pratiques décrites dans ce document. Si vous n'acceptez pas cette politique, veuillez cesser d'utiliser l'application et contacter votre administrateur RH.

> **Important :** Cette application est destinée à un usage professionnel uniquement. Elle est déployée par votre employeur dans le cadre d'une relation contractuelle. Les données collectées sont traitées à la fois par Batimemo en tant qu'éditeur de la solution, et par votre employeur en tant que responsable de traitement.

## 2. Données collectées

### 2.1 Données d'identification personnelle

| Donnée | Source |
|--------|--------|
| Prénom et nom | Compte employé créé par l'administrateur RH |
| Adresse e-mail professionnelle | Compte employé |
| Numéro de téléphone (principal et secondaire) | Profil employé |
| Photo de profil / Avatar | Téléversée par l'utilisateur |
| Code employé | Assigné par l'employeur |
| Poste / Désignation | Profil employé |
| Statut d'emploi | Profil employé |
| Genre | Profil employé |
| Adresse postale | Profil employé |

### 2.2 Données de localisation

| Donnée | Finalité |
|--------|----------|
| Coordonnées GPS précises (latitude / longitude) | Pointage de présence géolocalisé |
| Altitude et précision de la localisation | Vérification de l'emplacement |
| Cap et vitesse de déplacement | Suivi des déplacements professionnels |
| Horodatage des positions | Historique de présence |
| Distance parcourue durant les heures de travail | Calcul des indemnités kilométriques |
| Statut de connectivité réseau (WiFi / réseau mobile) | Détection du lieu de travail |

> La localisation peut être collectée **en arrière-plan** lorsque la fonctionnalité de pointage automatique est activée par votre employeur.

### 2.3 Données biométriques

| Donnée | Finalité |
|--------|----------|
| Données d'empreinte digitale | Authentification biométrique locale (stockée sur l'appareil, non transmise) |
| Données de reconnaissance faciale | Pointage de présence par détection du visage (via Google ML Kit Face Detection) |

> Les données biométriques d'empreinte digitale ne quittent jamais votre appareil. Elles sont traitées exclusivement par le système d'exploitation (Android / iOS). Les données de reconnaissance faciale sont traitées localement sur l'appareil pour la vérification de présence.

### 2.4 Données de l'appareil

| Donnée | Finalité |
|--------|----------|
| Identifiant unique de l'appareil (Device UID) | Sécurité et contrôle d'accès |
| Marque, modèle et version du système d'exploitation | Compatibilité et support technique |
| Adresse IP publique | Journalisation de sécurité |
| Niveau de batterie | Surveillance des appareils en mode terrain |
| État de la connectivité réseau | Synchronisation des données hors ligne |
| Données des capteurs de mouvement (accéléromètre) | Détection du mode de déplacement (marche / transport) |

### 2.5 Données professionnelles et RH

| Catégorie | Données concernées |
|-----------|-------------------|
| **Présence** | Pointages d'entrée/sortie, historique de présence, demandes de régularisation |
| **Congés** | Demandes de congés, soldes, types de congés, approbations |
| **Notes de frais** | Montants, catégories, justificatifs (photos de reçus), statuts d'approbation |
| **Paie** | Fiches de paie, structure salariale, déductions, ajustements |
| **Prêts** | Demandes de prêt, montants, échéanciers de remboursement |
| **Actifs** | Équipements assignés, état des actifs, signalements de problèmes |
| **Documents** | Demandes de documents, documents téléversés, dates d'expiration |
| **Disciplinaire** | Avertissements reçus, appels, accusés de réception |
| **Politiques RH** | Accusés de lecture des politiques de l'entreprise |
| **Projets** | Affectations de projets, feuilles de temps, heures travaillées, taux de facturation |
| **Calendrier** | Jours fériés, événements d'équipe, absences planifiées |
| **Tableau d'affichage** | Annonces lues, notifications reçues |

### 2.6 Données médias

| Donnée | Finalité |
|--------|----------|
| Photos prises par la caméra | Justificatifs de notes de frais, documents, pointage facial |
| Images sélectionnées depuis la galerie | Pièces justificatives |

| Enregistrements audio | Mises à jour de tâches de projet (si activé) |
| Vidéos | Preuves de terrain (si activé par l'employeur) |

### 2.7 Données d'utilisation et analytiques

| Donnée | Outil |
|--------|-------|
| Événements d'utilisation de l'application | Firebase Analytics (Google) |
| Rapports de plantage et erreurs | Firebase Crashlytics (Google) |

| Journaux des requêtes API | Serveur Batimemo (usage interne uniquement) |

## 5. Partage des données avec des tiers

Nous ne vendons pas vos données personnelles. Vos données peuvent être partagées avec les tiers suivants uniquement :

### 5.1 Google (Firebase)
- **Firebase Cloud Messaging** : Envoi de notifications push
- **Firebase Analytics** : Analyse anonymisée de l'utilisation de l'application
- **Firebase Crashlytics** : Rapports de plantage et de performance
- **Politique de confidentialité Google :** https://policies.google.com/privacy

### 5.2 Serveur Batimemo
- Toutes les données de l'application sont synchronisées avec nos serveurs hébergés à l'adresse : `https://batimemo.com/api/V1/`
- L'accès est sécurisé par HTTPS (TLS 1.2+) et authentification par token

### 5.3 Votre employeur (Administrateur RH)
- Votre employeur a accès à vos données de présence, localisation, congés, feuilles de temps et toutes les données RH vous concernant via le back-office Open Core HR
- Les données de localisation et de mouvement sont explicitement partagées avec votre responsable RH conformément à la finalité de l'application

### 5.4 Google Maps Platform
- Utilisé pour l'affichage cartographique des pointages géolocalisés
- **Politique Google Maps :** https://maps.google.com/help/terms_maps/

## 7. Droits des utilisateurs

Conformément au RGPD et aux législations applicables sur la protection des données, vous disposez des droits suivants :

- **Droit d'accès :** Obtenir une copie de vos données personnelles traitées
- **Droit de rectification :** Corriger des données inexactes ou incomplètes
- **Droit à l'effacement :** Demander la suppression de vos données (sous réserve des obligations légales de conservation)
- **Droit à la limitation du traitement :** Restreindre certains traitements de vos données
- **Droit à la portabilité :** Recevoir vos données dans un format structuré et lisible par machine
- **Droit d'opposition :** Vous opposer à certains traitements fondés sur l'intérêt légitime
- **Droit de retirer votre consentement :** Pour les traitements basés sur le consentement (localisation, biométrie, notifications)

> **Comment exercer vos droits :** Contactez votre administrateur RH ou envoyez votre demande à [privacy@batimemo.com](mailto:privacy@batimemo.com). Nous répondrons dans un délai de **30 jours**.

> Vous avez également le droit d'introduire une réclamation auprès de l'autorité de protection des données compétente dans votre pays.

## 8. Sécurité des données

Nous mettons en œuvre les mesures techniques et organisationnelles suivantes pour protéger vos données :

- **Chiffrement en transit :** Toutes les communications entre l'application et nos serveurs utilisent HTTPS (TLS 1.2 minimum)
- **Authentification sécurisée :** Authentification par token (JWT), option d'authentification biométrique locale
- **Stockage local chiffré :** Les données mises en cache localement sur l'appareil sont stockées dans une base Hive avec isolation des données par utilisateur
- **Contrôle d'accès :** Accès aux données RH limité selon le rôle de l'utilisateur (employé, manager, RH, admin)
- **Surveillance des incidents :** Firebase Crashlytics surveille les anomalies techniques en temps réel
- **Identifiant appareil :** Lié à chaque session pour détecter les accès non autorisés

## 12. Informations spécifiques aux plateformes

### Google Play Store (Android)

L'application utilise les permissions Android suivantes :

| Permission | Justification |
|-----------|---------------|
| `CAMERA` | Reconnaissance faciale pour le pointage de présence et prise de photos de justificatifs |
| `ACCESS_FINE_LOCATION` | Pointage géolocalisé précis (GPS) |
| `ACCESS_COARSE_LOCATION` | Localisation approximative en complément |
| `POST_NOTIFICATIONS` | Alertes RH, rappels de pointage, approbations |
| `READ_EXTERNAL_STORAGE` | Sélection de fichiers comme pièces justificatives |
| `WRITE_EXTERNAL_STORAGE` | Téléchargement de fiches de paie et documents |
| `RECORD_AUDIO` | Enregistrement de mises à jour audio sur les tâches de projet |
| `VIBRATE` | Retour haptique pour les notifications |

### Apple App Store (iOS)

| Clé de permission | Usage déclaré |
|-------------------|---------------|
| `NSCameraUsageDescription` | Partage de mises à jour et pointage de présence avec reconnaissance faciale |
| `NSLocationAlwaysAndWhenInUseUsageDescription` | Suivi de présence et calcul des distances parcourues |
| `NSLocationWhenInUseUsageDescription` | Localisation pour le pointage de présence |
| `NSMotionUsageDescription` | Détection du mode de déplacement (marche / transport) — partagé avec RH uniquement |
| `NSPhotoLibraryUsageDescription` | Sélection d'images comme preuves de notes de frais |
